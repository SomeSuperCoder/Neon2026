# PoH Blockchain CLI Usage Guide

This document describes how to use the CLI tool for file-based state operations.

## Building the CLI

```bash
go build -o poh-blockchain ./cmd/main.go
```

## Commands

### 1. Account Creation

Create a new account with an initial balance:

```bash
./poh-blockchain account create --balance <amount> --output <keypair-file> --state <db-path>
```

**Default paths:**
- Output: `data/keypair.json`
- State DB: `data/state.db`

**Example:**
```bash
./poh-blockchain account create --balance 1000000 --output data/my_account.json
```

This command:
- Generates a new Ed25519 keypair
- Creates an account file in the state database
- Saves the keypair to a JSON file for later use
- Automatically creates the `data/` directory if it doesn't exist

**Output:**
```
Account created successfully!
Address: 8da2721607fa4cfc2f174941331a2cc5ef02d798af42a7920580074905687722
Balance: 1000000
Keypair saved to: my_account.json
```

### 2. Balance Transfer

Transfer balance from one account to another:

```bash
./poh-blockchain transfer --from <keypair-file> --to <destination-address> --amount <amount> --state <db-path>
```

**Default state DB:** `data/state.db`

**Example:**
```bash
./poh-blockchain transfer --from data/my_account.json --to cc63bd32367830bf8a2f66390decee3f75bfdc844dcf0d6291e0be6c5114f93b --amount 50000
```

This command:
- Loads the source account keypair
- Creates and signs a transfer transaction
- Deducts the transaction fee from the source account
- Transfers the specified amount to the destination

**Output:**
```
Transfer successful!
Transaction ID: eec87212b62be415d2c094cdd5205cf5bce78a2092273ada790982515f155a51
Amount: 50000
From: 8da2721607fa4cfc2f174941331a2cc5ef02d798af42a7920580074905687722
To: cc63bd32367830bf8a2f66390decee3f75bfdc844dcf0d6291e0be6c5114f93b
```

### 3. Account Query

Query account information:

```bash
./poh-blockchain query --address <account-address> --state <db-path>
```

**Default state DB:** `data/state.db`

**Example:**
```bash
./poh-blockchain query --address 8da2721607fa4cfc2f174941331a2cc5ef02d798af42a7920580074905687722
```

**Output:**

### 4. Transaction Submission

Submit a pre-signed transaction from a JSON file:

```bash
./poh-blockchain submit --tx <transaction-file> --state <db-path>
```

**Default state DB:** `data/state.db`

**Example:**
```bash
./poh-blockchain submit --tx my_transaction.json
```

This command:
- Loads a transaction from a JSON file
- Validates and processes the transaction
- Returns the transaction ID and gas used

**Note:** The transaction file must contain a properly formatted and signed transaction in JSON format.

### 5. Transaction Status

Check the status of a transaction:

```bash
./poh-blockchain status --tx <transaction-id> --state <db-path>
```

**Default state DB:** `data/state.db`

**Example:**
```bash
./poh-blockchain status --tx eec87212b62be415d2c094cdd5205cf5bce78a2092273ada790982515f155a51
```

**Output:**
```
Transaction Status Check:
  Transaction ID: eec87212b62be415d2c094cdd5205cf5bce78a2092273ada790982515f155a51
  Note: Transaction history tracking not yet implemented
  State database: data/state.db (accessible)
```

**Note:** Full transaction history tracking is not yet implemented. This command currently only verifies database accessibility.

### 6. Help

Display help information:

```bash
./poh-blockchain help
```

## Keypair File Format

The keypair file generated by `account create` has the following JSON format:

```json
{
  "address": "8da2721607fa4cfc2f174941331a2cc5ef02d798af42a7920580074905687722",
  "public_key": "8da2721607fa4cfc2f174941331a2cc5ef02d798af42a7920580074905687722",
  "private_key": "76610c1be43370ae2f1b5909a69ba1c359eb5fb6b6cb4d0acf2fd21aed527ad68da2721607fa4cfc2f174941331a2cc5ef02d798af42a7920580074905687722"
}
```

**Security Note:** Keep your keypair files secure! The private key allows full control over the account.

## Transaction Fees

Transaction fees are calculated based on:
- Base fee: 5000 units
- Per instruction: 1000 units
- Per signature: 500 units

For a simple transfer (1 instruction, 1 signature):
- Total fee: 5000 + 1000 + 500 = 6500 units

## Running as a Node

To run the blockchain node (not CLI mode):

```bash
./poh-blockchain --type <leader|replica> --port <port> [options]
```

**Example:**
```bash
# Run as leader
./poh-blockchain --type leader --port 8080 --db leader.db

# Run as replica
./poh-blockchain --type replica --port 8081 --peers localhost:8080 --db replica.db
```

## Complete Example Workflow

```bash
# 1. Create two accounts (files will be saved in data/ directory)
./poh-blockchain account create --balance 10000000 --output data/alice.json
./poh-blockchain account create --balance 5000000 --output data/bob.json

# 2. Get Bob's address from the keypair file
BOB_ADDR=$(cat data/bob.json | grep address | cut -d'"' -f4)

# 3. Transfer from Alice to Bob
./poh-blockchain transfer --from data/alice.json --to $BOB_ADDR --amount 1000000

# 4. Query both accounts to verify
ALICE_ADDR=$(cat data/alice.json | grep address | cut -d'"' -f4)
./poh-blockchain query --address $ALICE_ADDR
./poh-blockchain query --address $BOB_ADDR
```

## Data Directory

All blockchain data is stored in the `data/` directory by default:
- `data/state.db/` - BadgerDB state database
- `data/*.json` - Keypair files

This directory is gitignored to prevent accidentally committing sensitive keypair data or large database files.

## Troubleshooting

### "file not found" error
- Ensure the state database path is correct
- For the first operation, the database will be created automatically

### "insufficient balance" error
- Check the account balance with the `query` command
- Ensure the account has enough balance to cover both the transfer amount and transaction fees

### "invalid address" error
- Verify the address is a valid 64-character hexadecimal string
- Addresses are case-sensitive

### "failed to load keypair" error
- Ensure the keypair file exists and is readable
- Verify the JSON format is correct
